React + TypeScript 기반으로 “사과 합 10 멀티플레이 웹게임”을 구현해줘.

목표:
apple_box.js의 코드를 참고하여 드래그한 범위 내에 있는 사과에 표시되어있는 숫자들의 합이 10인 경우 해당 사과들을 없애는 게임을 멀티플레이 기반 플랫폼으로 재 구축한다.

필수 기능:
1. 시작 화면
- 닉네임 입력
- “방 만들기” 버튼
- “방 참여하기” 버튼
- 방 참여 시 방 코드를 입력할 수 있게 한다.

2. 방 만들기
- 랜덤 방 코드를 생성한다.
- 방장은 게임 설정을 바꿀 수 있다.
- 방 코드를 화면에 표시해서 다른 사람이 참여할 수 있게 한다.

3. 방 참여
- 사용자가 방 코드를 입력하면 해당 방에 들어간다.
- 대기방에서 참가자 목록을 실시간으로 보여준다.

4. 게임 설정
- 총 몇 판을 할지 설정할 수 있어야 한다. 예: 1판, 3판, 5판
- 리더보드 계산 방식을 선택할 수 있어야 한다.
  - sum: 모든 판 점수 합계
  - best: 여러 판 중 최고 점수 하나

5. 게임 시작
- 방장이 시작 버튼을 누르면 모든 참가자에게 동시에 게임이 시작된다.
- 같은 room seed와 round index를 사용해서 모든 참가자가 같은 사과 배치를 플레이해야 한다.
- Math.random()을 직접 쓰지 말고 seed 기반 pseudo random generator를 구현해서 사용한다.

6. 게임 화면
- 720x470 비율의 게임 영역을 만든다.
- 여러 개의 사과를 배치한다.
- 각 사과에는 1~9 숫자를 표시한다.
- 마우스 드래그와 모바일 터치 드래그를 모두 지원한다.
- 드래그 영역 안에 포함된 사과 숫자 합이 10이면 제거한다.
- 점수는 제거된 사과 개수 또는 제거 성공 횟수 기준으로 계산하되, 코드에서 쉽게 바꿀 수 있게 상수화한다.
- 제한 시간 타이머를 둔다. 기본값은 120초로 한다.

7. 라운드 진행
- 한 판이 끝나면 해당 라운드 점수를 저장한다.
- 설정된 N판이 끝날 때까지 다음 라운드를 진행한다.
- 각 라운드는 seed + roundIndex 방식으로 서로 다른 판을 생성한다.

8. 리더보드
- 게임이 끝나면 참가자별 라운드 점수와 최종 점수를 보여준다.
- sum 모드에서는 모든 라운드 점수 합계로 정렬한다.
- best 모드에서는 가장 높은 단일 라운드 점수로 정렬한다.

9. 실시간 동기화
- Firebase Realtime Database 또는 Supabase Realtime 중 하나를 사용해 구현한다.
- 환경변수 기반으로 API 키를 관리한다.
- .env.example 파일을 추가한다.
- README에 설정 방법을 적는다.
- 로컬 개발 환경에서 실행 가능한 상태로 만든다.

10. 코드 품질
- 컴포넌트를 적절히 분리한다.
- TypeScript 타입을 명확히 정의한다.
- seed random, room code 생성, leaderboard 계산 로직은 별도 utils로 분리한다.
- 주요 로직에 간단한 테스트를 추가한다.
- ESLint/Prettier 설정이 있다면 기존 설정을 따른다.
- 없다면 최소한 npm run build가 통과하도록 한다.
